Angular Version-5 It was released in November 2017

This release focused on making Angular application to reduce the bundle size and faster to use.

It supports TypeScript 2.5

Angular Version 5 Features

Http Deprecated and HttpClient recommended to use

The angular/http module was used for making HTTP requests in Angular applications. The Angular team has now deprecated Http in version 5. The HttpClient API from angular/common/http package that is now recommended for use.

{{this.angular5Data[0].version5[0].httpclient}}

Support for Multiple Export Alias

In angular we could only export items individually from a module using their original names this could lead to lengthy or repetitive imports in other components, especially when using components with long or similar names. Export aliases offered a way to create shorter or more convenient alternatives names to imports.

{{this.angular5Data[0].version5[1].alias}}

Internationalized Number, Date, and Currency Pipes

Angular 5 provides several built-in pipes designed to format numbers, dates, and currencies according to user locale settings and it increase standardization across browsers and eliminate the need for i18n polyfills

To provide locale support and configurations for old pipe before angular5 we need to import the DeprecatedI18NPipesModule after the CommonModule

{{this.angular5Data[0].version5[2].localeSupport}}

Improved Decorator Support

It supports to reduce the expression for lambdas, and the value of useValue, useFactory, and data in object literals lambda can be used instead of a named function 

{{this.angular5Data[0].version5[3].decSupport}}

Build Optimization

Angular team focused on making Angular 5 faster, smaller and easier to use, production builds created with the Angular CLI it will apply the build optimizer by default and it will removes Angular decorators from your app's runtime codes thereby reducing the size of your bundle and increasing the boot speed of your application

Angular Universal State Transfer API and DOM Support

Angular Universal is a project focused on helping developers to perform server side rendering and rendering the Angular applications on the server and then bootstrapping on top of the generated HTML

Two modules have been adde to the Angular Universal ServerTransferStateModule and BrowserTransferModule These modules allow to generate information as part of rendering with platform-server and then transfer it to the client side to avoid re-generation of the same information

Faster Compiler

Improved the Angular compiler to support incremental compilation. This provides faster rebuilds, especially for production builds and builds with AOT. also added features to our Decorators and made it possible to ship smaller bundles by removing whitespace

 Angular Version 6 Features

Angular Elements

The Angular Elements package will gives the ability to create an Angular component and then use that component as a web component which can be used in any HTML page (even if that page is not using the Angular framework) in other environments. It actually takes an Angular component and then wraps it within a custom element, such as a DOM element, so that we can use our Angular component in other projects which do not use Angular.

ng update & ng add commands

ng update: Updates Angular CLI and project packages to newer versions.

ng add: Integrates third-party libraries and schematics into our project with pre-defined configurations.

Service Providers within Service

Previously, we are using app.module.ts providers to declare the service and use as singleton in througth the Application

Angular 6 allowed defining providers directly within the service using providedIn: 'root' and make them as a singleton

{{this.angular6Data[0].version6[0].service}}

ng-template instead of template directive

The "template" element was depreciated that was launched in Angular 4 Instead of using "template", now we need to use "ng-tempalate" It offering better integration with structural directives like *ngFor and *ngIf

{{this.angular6Data[0].version6[1].ngTemp}}

Ivy: New Rendering Engine

In Angular 6 introduced their third rendering engine called Ivy. Ivy is the next generation Angular rendering engine In previous versions of Angular view engine for rendering purposes

Angular compiles our templates into equivalent TypeScript code and then that TypeScript code is compiled to JavaScript and then the result is rendered to users. So Ivy renderer is the new rendering engine which is basically designed to support backward compatibility with existing renderers and then also focused to improve the speed of rendering and it also optimized the size of the final package In Angular, it will not be the default renderer but we can manually enable it using the compiler options.

Service Worker Support

Service workers are basically scripts which run in the web browser and manage to cache an application. Service workers were first introduced in Angular 5. In Angular 6, service workers come with some bug fixes, including some new functionalities. So when we deploy the latest version of the application, we may need to deactivate or uninstall the existing service workers. In Angular 5, there is no straightforward option for doing this, but Angular 6 brings this functionality with the new script file named safety-worker.js, which is actually a part of the production bundle which helps us to unregister the existing service worker
