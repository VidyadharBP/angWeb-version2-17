 Angular Version 2 Features

Components

Components are the logical piece of code contains the bussiness logic.

Each component consist of

Template -> Contains View displayed on the Dom.

Class -> Contains the bussiness logic properties and methods and this will support the View

MetaData -> It tells about extra information about the component like template,style etc

Modules

Modules are the building block of application they contains the information of Components,Service and if they are using other module then there information

Every application consist of at least one module that is Root Module that bootstraps our application

we have Feature modules These modules they represent features or functionalities within our application They import necessary components, directives, services, and pipes specific to that feature

Directives

Directives are the custom html element they are used to add or remove the element and we can make the change on the element

Directives will power up the html

*ngIf

The ngIf will used to add or remove the element from the Dom it will add the element if its get true value else it will remove the element

*ngFor

Data Display

To display the value of the properties of the class in the HTML

We will use

[value] = ”username” This is used to bind the username to the input element value property.

(input) = ”expression” This is a way of binding an expression to the input elements input event

$event An expression exposed in event bindings by Angular, which has the value of the events payload

Forms

Angular 2 we can also design forms which use two-way binding by using the ngModel directive

The ngModel directive is used to bind the object of the class to the separate elements on the form.

Services

Services are mainly used for when a common functionality needs to be provided to a various Components or modules

First we need to create a Service class then we need to import that in the providers array of app.module.ts

After that in which component we need that service we need to inject that in the constructors

{{this.angular2Data[0].ver2Component[6].service}}

Routing

Routing helps user to navigate one page to another page

For routing we will create routes array and it contains objects with path and component like witch path what component need to navigate

we need to provide space to open routed component for that we need to use "router-outlet" element provided by Angular

we need to use "routerLink" to give link of paths provided by the Angular

Angular 4 is almost the same as Angular 2. Angular 2 application will work without any issues with Angular 4

Why Angular4 and Not Angular3?

The Angular team faced some versioning issues internally with their modules and due to the conflict they had to move on and release the next version of Angular the Angular4.

Angular4 uses TypeScript 2.2 version
